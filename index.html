FPGA 4x4 Keypad Integration: (Verilog Programing)

Integrated a 4x4 Keypad to an FPGA in Verilog code. An FPGA is similar to that of a microcontroller, but rather than having the processor jump from one part of the code to another simulating parallelism, an FPGA is where all the code runs in parallel simultaneously. The 4x4 keypad is setup with 8 total pins, 4 Rows & 4 Columns. The 4 row pins were connected to I/O pins on the FPGA as input PULLDOWN pins, and the 4 column pins were connected as output PULLUP pins. Therefor as each column went high and a button on the keypad was pressed the voltage outputting from the FPGA would flow through the closed button and back into the input pins on the FPGA. Using case statements to recognize which row and column was pressed, I could display the correct number or symbol on the 7 Segment LED display

Best path end-to-end in a 233-Hexagon array: (C++ Programing)

Developed a C++ program that reads a text file with 233 random numbers ranging from 1-999. The program would then place a number inside each of the 233 hexagons correspondingly. Fan alteration by heat detection: (C Programing)

Explored the capabilities of a PIC Microcontroller to alter a fans speed based on the temperature of a thermometer sensor. Created a Wheatstone bridge that incorporated a thermometer on one of the legs and an interweaved an Op-Amp. The output of the Op-Amp was used as the input for the PIC microcontroller. The input that was coming in would be an analog signal and had to be converted to a Digital Signal. Used the on board ADC register to do this, and stored the value in a temporary onboard register ADRESH. Created case statements which compared ADRESH to a hard coded value which would alter the fan speed using Pulse Width Modulation.

Filtering an Audio File: (MATLAB Programing)

Explored Matlab’s capabilities of reading, writing and cleaning audio signals. Recorded audio ends up with unintentional residual low-level sound, “Audio Noise”. Luckily with post-processing development we were able to remove most noise and clean up the audio signal. An audio signal is a range of electrical voltage between 0 and VDD, which make up frequencies. To clean the audio signal, we found its impulse response, which is a finite discrete-time sequence, to observe any unwanted frequencies. Then we designed and used multiple FIR filters removing the unwanted frequencies. The four types of Linear-Phase FIR filters that we used were: low-pass filter, high-pass filter, band-pass filter, band-stop filter.

FIFO Memory Buffer: (Verilog Programing)

In a synchronous FIFO, data values are written sequentially into a memory array using a clock signal, and the data values are sequentially read out from the memory array using the same clock signal. I developed Verilog code to resemble this FIFO memory which included a memory array, write control logic, and read control logic. The memory array was implemented as a dual‐port read/write memory with simultaneous read and write accesses. I used a single clock port for both the read and write operation, date input & output registers, write & read enable inputs, and finally full, empty, almost full, and almost empty flags controlled using binary pointers.

Video Compression: (MATLAB Programing)

Using my knowledge of Video Compression, I developed a script to compress videos using motion compensation and discrete cosine transform (DCT) techniques. I Calculated motion vectors between successive frames and used them to minimize redundant information. I then divided each frame into smaller sub-frame matrices allowing for more precise compression on each individual sub-matrix. I then recombined all sub-matrices back together and finally applied a quantization technique to achieve further compression. Bellow is the Step-By-Step process in which the compression occurred. Splitting the video into frames, RGB to YCbCr conversion of frames, 4:4:4 to 4:2:0 conversion, Motion Estimation and vectoring, DCT, Quantize, Zig Zag and RLE ordering, Encoding, Decoding, DeQuantization, IDCT, Reconstruct I and P Frames, and combine frames into compressed video.
